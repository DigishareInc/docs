L'accès sécurisé à l'API Digishare est obligatoire. Nous prenons en charge deux méthodes principales d'authentification selon vos besoins d'intégration.

::note
Toutes les requêtes API doivent être effectuées via **HTTPS**. Les appels effectués en HTTP simple échoueront.
::

## 1. Jeton d'Accès Personnel

Le **Jeton d'Accès Personnel (Bearer Token)** est idéal pour les intégrations serveur-à-serveur, les scripts, ou lorsque vous construisez une intégration pour votre propre organisation.

::steps

### Générer le Jeton

Accédez à **Paramètres > Jetons d'Accès Personnels** dans votre tableau de bord Digishare et cliquez sur **Générer un Nouveau Jeton**.

### Copier le Jeton

Copiez le jeton immédiatement. Pour des raisons de sécurité, il ne sera **pas** affiché à nouveau.

### Authentifier

Incluez le jeton dans l'en-tête `Authorization` de vos requêtes HTTP.
::

### Essayez-le

::api-playground{method="GET" url="https://api.digishare.ma/v1/campaigns" description="Listez toutes vos campagnes en utilisant l'authentification par Jeton Bearer." :variables='{"token": "VOTRE_JETON_ACCES_PERSONNEL"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}'}
::

## 2. Identifiants Client OAuth 2.0

Utilisez le **Flux d'Identifiants Client** si vous construisez une application tierce ou un service qui agit en son propre nom.

::steps

### Obtenir les Identifiants

Obtenez votre `Client ID` et `Client Secret` depuis la page **Développeur > Applications**.

### Demander le Jeton

Échangez vos identifiants contre un jeton d'accès via le point de terminaison `/oauth/token`.

### Utiliser le Jeton

Utilisez le `access_token` retourné dans l'en-tête `Authorization` pour les requêtes suivantes.
::

### Demande de Jeton

::api-playground{method="POST" url="https://api.digishare.ma/v1/oauth/token" description="Échangez les identifiants client contre un jeton d'accès." :headers='{"Content-Type": "application/json"}' :body='{"grant_type": "client_credentials", "client_id": "VOTRE_CLIENT_ID", "client_secret": "VOTRE_CLIENT_SECRET"}'}
::

### Réponse du Jeton

```json
{
  "token_type": "Bearer",
  "expires_in": 31536000,
  "access_token": "VOTRE_ACCESS_TOKEN"
}
```

::warning
**Meilleure Pratique de Sécurité :** N'exposez jamais votre `Client Secret` ou `Jeton d'Accès Personnel` dans le code côté client (navigateurs). Ils ne doivent être utilisés que dans des environnements serveur sécurisés.
::
