## 🚀 Référence Rapide : Erreurs API Courantes

| Statut HTTP | Erreur / Code | Signification | Action Recommandée |
| :--- | :--- | :--- | :--- |
| **400 Bad Request** | `sender_label_id invalid` | Le `sender_label_id` fourni n'existe pas ou est inactif. | Vérifiez votre identifiant d'expéditeur dans le tableau de bord Digishare. |
| **402 Payment Required** | `Itinarea balance not enough` | Le solde de votre entreprise est insuffisant pour l'envoi de SMS. | Rechargez votre solde ou votre quota SMS. |
| **404 Not Found** | `Campaign not found` | L'ID de campagne demandé n'existe pas. | Vérifiez le paramètre d'ID de campagne. |
| **422 Unprocessable** | `The Campaign requires a valid Message Template` | Absence de `message_template` et de `message_template_id`. | Fournissez soit un objet `message_template` en ligne, soit un `message_template_id`. |
| **429 Too Many Requests** | `rate_limited` | Limite de débit API atteinte. | Implémentez un temporisateur d'attente (backoff) et regroupez vos destinataires. |

## 🔍 Diagnostic des SMS et Journaux de Livraison

Lorsqu'une campagne est traitée, les journaux d'acheminement de chaque message sont enregistrés dans le système.

### Consulter les Raisons d'Échec de Livraison dans le Tableau de Bord

1. Accédez à la page de détail de votre campagne (`/campaigns/:id`).
2. Faites défiler jusqu'à la section **Rapport de Livraison SMS**.
3. Cliquez sur **Détails Complets** pour inspecter l'état de livraison de chaque destinataire :
   - **`FAILED`** : Message rejeté par la passerelle ou l'opérateur télécom.
   - **`DELIVERED`** : Remis avec succès au téléphone du destinataire.
   - **`EXPIRED`** : Téléphone éteint ou hors de portée au-delà du délai de rétentative de l'opérateur.

## 🚨 Dépannage des Problèmes Courants

### 1. Exception de Solde Insuffisant

```json
{
  "message": "Itinarea balance not enough",
  "status_code": 402,
  "errors": {
    "company_id": 1,
    "channel_id": "sms"
  }
}
```

**Résolution** : Assurez-vous que votre entreprise dispose de crédits actifs attribués au canal `sms` ou d'un solde global non restreint.

### 2. Format de Numéro Mobile Invalide

Si le numéro de téléphone d'un destinataire est mal formé (ex. indicatif pays manquant ou chiffres invalides), l'initialisation de la campagne l'ignorera ou le marquera comme échoué lors de la création du journal de livraison.

**Résolution** : Normalisez tous les numéros au **format E.164** (ex. `212671791198`).
