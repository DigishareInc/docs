Ce guide explique comment envoyer une campagne WhatsApp avec des fichiers encodés en **base64** directement dans votre requête API, sans avoir à les télécharger séparément au préalable.

::note
**Prérequis** :
- Un **Bearer Token** valide dans l'en-tête Authorization.
- Un **Modèle de Message Digishare** (ITM ou **WABA TM**) configuré pour accepter un en-tête de fichier ou de média.
- Les données du fichier encodées sous forme de chaîne base64.
::

## 1. Fonctionnement

Au lieu de fournir l'URL publique d'un fichier, vous pouvez envoyer le fichier encodé en base64 dans un bloc spécial `base64_files` à la racine de votre payload.

1. Définissez un nom d'espace réservé personnalisé (placeholder), par exemple `PLACEHOLDER_MA_FACTURE`.
2. Mappez ce placeholder dans votre bloc `global_data` (ou dans les champs individuels des `recipients`) là où le modèle attend l'en-tête du média.
3. Définissez les détails du fichier (chaîne base64, nom du fichier, extension) sous le nom du placeholder dans l'objet `base64_files`.
4. Digishare traite automatiquement les fichiers base64, les télécharge dans la bibliothèque de stockage, génère le modèle et remplace toutes les occurrences du placeholder par l'ID du média correspondant.

## 2. API Implementation

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Créez une campagne WhatsApp avec des fichiers base64 intégrés.
variables:
  token: VOTRE_TOKEN
  senderId: VOTRE_SENDER_ID
  templateId: VOTRE_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Campagne de Distribution de Factures
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    wa_id: "%recipient.wa_id%"
    header:
      link: "PLACEHOLDER_FACTURE"
  recipients:
    - wa_id: "212600000000"
  base64_files:
    PLACEHOLDER_FACTURE:
      base64: "data:application/pdf;base64,JVBERi0xLjQK..."
      file_name: "facture_12345"
      extension: "pdf"
---
::

## 3. Champs de Configuration

L'objet `base64_files` est une map où chaque clé est votre chaîne de placeholder, et la valeur est un objet contenant :

| Champ | Type | Statut | Description |
| :--- | :--- | :--- | :--- |
| `base64` | `string` | **Requis** | La chaîne encodée en base64 représentant le fichier (peut inclure le préfixe du schéma Data URI). |
| `file_name` | `string` | Optionnel | Le nom du fichier à enregistrer (sans l'extension). |
| `extension` | `string` | Optionnel | L'extension du fichier (par exemple, `pdf`, `png`, `jpg`). |
