Ce guide explique comment envoyer une campagne WhatsApp incluant un **document PDF**.

::note
**Prérequis** :
- Un **Bearer Token** valide dans l'en-tête Authorization.
- Un **Modèle de Message Digishare** (ITM ou **WABA TM**) configuré pour accepter un en-tête avec lien PDF.
- Une URL PDF accessible publiquement.
::

## 1. Exigences de la Charge Utile

Pour envoyer un PDF personnalisé, vous devez fournir le lien de l'en-tête et les informations du destinataire via les blocs `global_data` et `recipients`.

### Champs Requis
- `wa_id` : Numéro de téléphone du destinataire (dans `global_data` comme variable de modèle).
- `header.link` : URL accessible du PDF.
- `tags` : Tableau optionnel pour catégoriser les campagnes.
- `conversation_id` : Identifiant optionnel pour le suivi des conversations.

## 2. Implémentation API (PDF Personnalisé)

Pour envoyer un document PDF unique et personnalisé à chaque destinataire (par exemple, une facture spécifique), utilisez une variable dynamique dans `global_data` et spécifiez l'URL du fichier pour chaque destinataire dans le tableau `recipients` :

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Envoyez des documents PDF personnalisés et uniques à chaque destinataire.
variables:
  token: VOTRE_TOKEN
  senderId: VOTRE_SENDER_ID
  templateId: VOTRE_ID_MODELE
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Distribution de Factures Personnalisées
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    wa_id: "%recipient.wa_id%"
    header:
      link: "%recipient.pdf_url%"
  recipients:
    - wa_id: "212600000000"
      pdf_url: "https://votre-site.com/factures/fact_ahmed.pdf"
    - wa_id: "212600000001"
      pdf_url: "https://votre-site.com/factures/fact_fatima.pdf"
  tags:
    - "campagne"
  conversation_id: "message-out"
---
::

## 3. Alternative : Gestion des PDF Statiques

Si votre campagne utilise le **même** document PDF pour tous les destinataires (par exemple, un catalogue produit ou une brochure générale), vous n'avez pas besoin de personnaliser l'URL par destinataire. Définissez simplement le lien du PDF en dur dans `global_data` :

```json
{
  "title": "Catalogue Mensuel",
  "channel": "whatsapp",
  "sender_label_id": "your-sender-label-id",
  "message_template_id": "your-message-template-id",
  "global_data": { 
    "wa_id": "%recipient.wa_id%",
    "header": {
      "link": "https://votre-site.com/catalogue.pdf"
    }
  },
  "recipients": [
    { "wa_id": "212600000000" },
    { "wa_id": "212600000001" }
  ],
  "tags": ["campagne"]
}
```
