Ce guide explique comment envoyer une campagne WhatsApp incluant un **bouton de lien externe**.

::note
**Prérequis** :
- **Jeton d'accès (Bearer Token)** valide.
- **Modèle de Message Digishare** (type ITM ou **WABA TM**) contenant un bouton de type `URL`.
::

## 1. Types de Liens

Digishare supporte deux types d'URLs dans vos modèles :
- **Statique** : Identique pour tous les destinataires.
- **Dynamique** : Le modèle contient une variable à la fin de l'URL (ex: `https://site.ma/id/{{1}}`).

## 2. Structure du Payload

Pour une URL dynamique, passez la valeur de la variable dans le champ `button0`.

::api-playground{method="POST" url="https://api.digishare.ma/v1/campaigns" description="Envoyer un bouton URL dynamique." :variables='{"token": "VOTRE_TOKEN", "senderId": "VOTRE_ID_EXPEDITEUR", "templateId": "VOTRE_ID_MODELE"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}' :body='{"title": "Paiement Commande", "channel": "whatsapp", "sender_label_id": "{senderId}", "message_template_id": "{templateId}", "recipients": [{"wa_id": "212600000000", "button0": "ORD-789"}] }'}
::

::warning
L'URL de base est définie lors de la création du modèle dans votre Dashboard. L'API ne permet d'envoyer que le **suffixe dynamique** (la variable).
::
