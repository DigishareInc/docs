Ce guide explique comment envoyer une campagne WhatsApp incluant un **document PDF**.

::note
**Prérequis** :
- **Jeton d'accès (Bearer Token)** valide.
- **Modèle de Message Digishare** (type ITM ou **WABA TM**) configuré pour accepter un PDF.
::

## 1. Préparation du Payload

Pour envoyer un PDF, vous devez fournir l'URL publique du fichier dans le champ `header_url`.

### Champs requis
- `wa_id` : Numéro du destinataire.
- `header_url` : URL directe du fichier PDF.
- `filename` : (Optionnel) Nom personnalisé du fichier affiché sur WhatsApp.

## 2. Exemple de Payload

::api-playground{method="POST" url="https://api.digishare.ma/v1/campaigns" description="Envoyer un PDF personnalisé." :variables='{"token": "VOTRE_TOKEN", "senderId": "VOTRE_ID_EXPEDITEUR", "templateId": "VOTRE_ID_MODELE"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}' :body='{"title": "Envoi Facture", "channel": "whatsapp", "sender_label_id": "{senderId}", "message_template_id": "{templateId}", "recipients": [{"wa_id": "212600000000", "header_url": "https://votre-site.ma/facture-123.pdf", "filename": "facture_janvier.pdf"}] }'}
::

## 3. Utilisation de `global_data` pour le PDF

Si le PDF est le même pour tout le monde (ex: un catalogue), placez-le dans `global_data`.

```json
{
  "title": "Catalogue Été",
  "global_data": { 
    "header_url": "https://cms.ma/catalogue.pdf", 
    "filename": "Catalogue_Digishare.pdf" 
  },
  "recipients": [
    { "wa_id": "212600000000" },
    { "wa_id": "212600000001" }
  ]
}
```
