Ce guide montre comment envoyer une campagne WhatsApp incluant un **code OTP** (One-Time Password).

::note
**Prérequis** :
- **Jeton d'accès (Bearer Token)** valide.
- **Modèle de Message Digishare** (type **WABA TM**) configuré pour l'authentification.
::

## 1. Fonctionnement des OTP

Pour les OTP via WhatsApp, vous devez utiliser un modèle de type **Authentication** (WABA TM) pour bénéficier des fonctionnalités d'autocomplétion sur Android/iOS.

### Champs requis
- `otp_code` : Le code de vérification généré par votre système.

## 2. Exemple de Payload

::api-playground{method="POST" url="https://api.digishare.ma/v1/campaigns" description="Envoyer un code OTP." :variables='{"token": "VOTRE_TOKEN", "senderId": "VOTRE_ID_EXPEDITEUR", "templateId": "VOTRE_ID_MODELE"}' :headers='{"Authorization": "Bearer {token}", "Content-Type": "application/json"}' :body='{"title": "Verification OTP", "channel": "whatsapp", "sender_label_id": "{senderId}", "message_template_id": "{templateId}", "recipients": [{"wa_id": "212600000000", "otp_code": "456789"}] }'}
::

::important
Le champ `otp_code` est réservé aux modèles d'authentification WABA TM officiels. Si vous utilisez un simple modèle de texte, utilisez une variable classique (`{{1}}`).
::
