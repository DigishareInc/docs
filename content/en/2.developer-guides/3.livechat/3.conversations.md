This document explains the core functionalities for managing and interacting with conversations via API. It covers creating conversations, checking for existing ones, and sending messages.

## 1. Create Conversation

Create a new conversation with a contact via an API provider. The system will intelligently resolve the contact and their platform identity, and manage any existing active or archived conversations according to the flags you provide.

**Endpoint**: `POST https://api.digishare.ma/v1/conversations/provider`

### Key Parameters / Payload

| Parameter                          | Type    | Required | Description                                                                            |
| :--------------------------------- | :------ | :------- | :------------------------------------------------------------------------------------- |
| `provider_instance_id`             | String  | Yes      | The ID of the API provider instance handling the channel.                              |
| `channel`                          | String  | Yes      | The messaging channel (e.g. `whatsapp`, `sms`).                                        |
| `recipient_id`                     | String  | Yes      | The phone number, WhatsApp ID, or platform-specific identifier of the contact.         |
| `platform_link_id`                 | String  | No       | ID of an existing PlatformLink, if known. Skips contact resolution if provided.        |
| `archive_active_conversation`      | Boolean | No       | Default: `true`. Archives the current active conversation before creating a new one.   |
| `reactivate_archived_conversation` | Boolean | No       | Default: `false`. Reactivates the most recently archived conversation.                 |

### Create Conversation Example

```bash
curl --location 'https://api.digishare.ma/v1/conversations/provider' \
  -H 'accept: application/json, text/plain, */*' \
  -H 'authorization: Bearer <JWT_TOKEN>' \
  -H 'content-type: application/json' \
  -H 'selectedcompanyid: <COMPANY_ID>' \
  --data-raw '{
    "provider_instance_id": "<PROVIDER_INSTANCE_ID>",
    "channel": "whatsapp",
    "archive_active_conversation": true,
    "reactivate_archived_conversation": false,
    "recipient_id": "<RECIPIENT_PHONE_NUMBER>",
    "third_id": "<THIRD_ID>"
  }'
```

### Execution Flow

1. **Validation**: Verifies the `provider_instance_id` exists.
2. **Contact Resolution**: Resolves the `PlatformLink`. If not found, creates a minimal `Third` contact record and a `PlatformLink` on the fly.
3. **Reactivation Check**: If `reactivate_archived_conversation` is `true`, the last archived conversation is reactivated. The currently active one is archived first if `archive_active_conversation` is also `true`.
4. **Archive Check**: If an active conversation exists and reactivation is not requested, archives it when `archive_active_conversation` is `true`.
5. **Creation**: Dispatches `CreateConversationAction` to create a new active conversation with the resolved template and contact.

## 2. Check Conversation

Query the system to see if a conversation already exists for a specific set of parameters. This helps retrieve the current state of a conversation and prevents duplicates.

**Endpoint**: `GET https://calclytics.digishare.ma/api/conversations/check`

### Query Parameters

All IDs should be provided in their encoded format.

| Parameter           | Type   | Description                                           |
| :------------------ | :----- | :---------------------------------------------------- |
| `company_id`        | String | Your Company Encoded ID.                              |
| `api_provider_id`   | String | The Encoded API Provider Instance ID.                 |
| `channel`           | String | The messaging channel (e.g. `whatsapp`, `sms`).       |
| `third_id`          | String | The encoded ID of the Third (contact).                |
| `channel_identity`  | String | The identity associated with the channel.             |
| `platform_link_id`  | String | The encoded PlatformLink ID *(Optional)*.             |

### Check Request Example

```bash
curl 'https://calclytics.digishare.ma/api/conversations/check?company_id=<COMPANY_ID>&api_provider_id=<PROVIDER_INSTANCE_ID>&channel=whatsapp&third_id=<THIRD_ID>&channel_identity=<CHANNEL_IDENTITY>' \
  -H 'sec-ch-ua-platform: "Windows"' \
  -H 'Authorization: Bearer <JWT_TOKEN>' \
  -H 'Referer: https://app.digishare.ma/' \
  -H 'User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)' \
  -H 'Accept: application/json, text/plain, */*' \
  -H 'sec-ch-ua-mobile: ?0'
```

### Check Response

Returns a JSON object detailing the discovery:

```json
{
  "success": true,
  "found": true, 
  "conversation": {
    "id": "...",
    "is_active": true,
    "last_message": { ... },
    "created_at": "..."
  },
  "identity_owner": "...",
  "timestamp": "2026-03-03T12:00:00Z"
}
```

## 2. Send Message

Send a message from an Agent (system user) to the contact in a conversation.

**Endpoint**: `POST https://api-app.digishare.ma/v1/event/conversation_message`

### Message Parameters / Payload

| Field                      | Type    | Description                                                                              |
| :------------------------- | :------ | :--------------------------------------------------------------------------------------- |
| `event_type`               | String  | Should be set to `conversation_message`.                                                 |
| `channel`                  | String  | The messaging channel (e.g. `whatsapp`).                                                 |
| `type`                     | String  | The type of the message (e.g. `text`).                                                   |
| `send_to_third`            | Boolean | Set to `true` to send the message to the external platform provider.                     |
| `api_provider_instance_id` | String  | The required ID of the API provider instance.                                            |
| `recipient_id`             | String  | The specific identifier of the recipient.                                                |
| `body`                     | String  | The actual content of the message.                                                       |
| `conversation_id`          | String  | Unique ID for the conversation *(Optional)*.                                             |

::warning
**24-Hour Guard**: If the elapsed time since the last inbound message exceeds 24 hours (e.g., WhatsApp's customer service window), delivery may be restricted by your provider.
::

### Message Request Example

```bash
curl --location 'https://api-app.digishare.ma/v1/event/conversation_message' \
--header 'Content-Type: application/json' \
--header 'Authorization: Bearer <JWT_TOKEN>' \
--data '{
    "event_type": "conversation_message",
    "channel": "whatsapp",
    "type": "text",
    "send_to_third": true,
    "api_provider_instance_id": "<PROVIDER_INSTANCE_ID>",
    "recipient_id": "<RECIPIENT_PHONE_NUMBER>",
    "body": "Message Body"
  }'
```

### Message Response (Success)

```json
{
    "event_id": "2633736349319958528",
    "status": "accepted",
    "timestamp": "2026-02-12T14:22:23.133414141Z"
}
```
