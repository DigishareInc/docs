This guide explains how to send a WhatsApp campaign with files encoded as **base64** strings directly in your API request, without needing to upload them separately beforehand.

::note
**Prerequisites**:
- Valid **Bearer Token** in Authorization header.
- **Digishare Message Template** (ITM or **WABA TM**) configured to accept a file or media header.
- The file data encoded as a base64 string.
::

## 1. How It Works

Instead of providing a public file URL, you can send the base64-encoded file inside a special `base64_files` block at the root of your payload. 

1. Define a custom placeholder name (e.g., `PLACEHOLDER_MY_FILE`).
2. Map this placeholder in your `global_data` (or individual `recipients` fields) where the template expects the media header. The placeholder must be referenced within a structured object:
   ```json
   "header": {
     "provider": "digishare",
     "id": "PLACEHOLDER_MY_FILE"
   }
   ```
3. Define the file details (base64 string, file name, extension) under the placeholder name in the `base64_files` object.
4. Digishare automatically processes the base64 files, uploads them to the media library, creates the template, and replaces all occurrences of the placeholder with the resulting media ID.

## 2. API Implementation

::api-playground
---
method: POST
url: https://api.digishare.ma/v1/campaigns
description: Create a WhatsApp campaign with embedded base64 files.
variables:
  token: YOUR_TOKEN
  senderId: YOUR_SENDER_ID
  templateId: YOUR_TEMPLATE_ID
headers:
  Authorization: Bearer {token}
  Content-Type: application/json
body:
  title: Invoice Distribution Campaign
  channel: whatsapp
  sender_label_id: "{senderId}"
  message_template_id: "{templateId}"
  global_data:
    wa_id: "%recipient.wa_id%"
    header:
      provider: "digishare"
      id: "PLACEHOLDER_INVOICE"
  recipients:
    - wa_id: "212600000000"
  base64_files:
    PLACEHOLDER_INVOICE:
      base64: "data:application/pdf;base64,JVBERi0xLjQK..."
      file_name: "invoice_12345"
      extension: "pdf"
---
::

## 3. Configuration Fields

The `base64_files` object is a map where each key is your placeholder string, and the value is an object containing:

| Field | Type | Status | Description |
| :--- | :--- | :--- | :--- |
| `base64` | `string` | **Required** | The base64-encoded string representation of the file (can include the Data URI scheme prefix). |
| `file_name` | `string` | Optional | The name of the file to save (without the extension). |
| `extension` | `string` | Optional | The file extension (e.g. `pdf`, `png`, `jpg`). |
