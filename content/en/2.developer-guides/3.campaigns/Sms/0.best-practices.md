## 1. SMS Character Limits & Character Encoding

Understanding SMS character encoding is critical to controlling campaign costs and avoiding unexpected message splitting.

### GSM 7-bit Encoding (Standard Text)

Basic Latin characters, numbers, and common punctuation use **GSM 7-bit** encoding:

- **Single SMS**: Up to **160 characters**.
- **Multi-part SMS**: Messages longer than 160 characters are split into segments of **153 characters** each (7 characters reserved for concatenation headers).

### Unicode / UCS-2 Encoding (Special Characters & Emojis)

If your message contains accents (`é`, `à`), non-Latin scripts (Arabic `مرحبا`, Cyrillic, etc.), or emojis (`🚀`):

- **Single SMS**: Up to **70 characters**.
- **Multi-part SMS**: Messages longer than 70 characters are split into segments of **67 characters** each.

::tip
**Cost Saving Tip**: Avoid emojis or non-GSM characters in high-volume promotional campaigns unless necessary, as a single emoji reduces the per-segment character limit from 160 to 70.
::

## 2. Phone Number Formatting (E.164 Standard)

Always format recipient phone numbers in standard **E.164 format** without spaces, dashes, or leading zeros:

- **Correct**: `"212671791198"` or `"+212671791198"`
- **Incorrect**: `"0671791198"`, `"212 671 791 198"`, `"+212 (0)671791198"`

```json
{
  "recipients": [
    { "mobile": "212671791198" },
    { "mobile": "212661234567" }
  ]
}
```

::note
The Digishare API automatically standardizes valid national numbers, but sending E.164 formatted strings guarantees faster processing and avoids delivery failures.
::

---

## 3. High-Volume Batching

For optimal API throughput and backend efficiency:

- Send recipients in batches of up to **1,000 items per API request** within the `recipients` array.
- Avoid firing single individual API calls for each recipient when sending broadcast campaigns.

---

## 4. Compliance & Opt-Out Requirements

- **Sender Label Compliance**: Ensure your `sender_label_id` is registered and approved in your region.
- **Opt-Out Instructions**: For promotional messages, include opt-out instructions if mandated by local regulations (e.g. *"Reply STOP to unsubscribe"*).
- **Time Window**: Respect local regulations regarding quiet hours for marketing SMS (e.g. avoiding late-night promotional broadcasts).
