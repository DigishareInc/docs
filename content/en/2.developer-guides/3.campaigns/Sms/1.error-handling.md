## 🚀 Quick Reference: Common API Error Responses

| HTTP Status | Error / Code | Meaning | Recommended Action |
| :--- | :--- | :--- | :--- |
| **400 Bad Request** | `sender_label_id invalid` | The provided `sender_label_id` does not exist or is inactive. | Verify your sender label ID in the Digishare dashboard. |
| **402 Payment Required** | `Itinarea balance not enough` | Your company balance is insufficient for SMS sending. | Recharge your account balance or SMS quota. |
| **404 Not Found** | `Campaign not found` | The requested campaign ID does not exist. | Check the campaign ID parameter. |
| **422 Unprocessable** | `The Campaign requires a valid Message Template` | Missing both `message_template` and `message_template_id`. | Provide either an inline `message_template` object or a `message_template_id`. |
| **429 Too Many Requests** | `rate_limited` | API rate limit exceeded. | Implement exponential backoff and batch recipient requests. |

## 🔍 SMS Delivery & Log Diagnostics

When a campaign is processed, individual message delivery logs are recorded in the system push logs.

### Checking Delivery Failure Reasons in the Dashboard

1. Navigate to your campaign detail page (`/campaigns/:id`).
2. Scroll to the **SMS Delivery Report** section.
3. Click **Full Details** to inspect individual recipient delivery statuses:
   - **`FAILED`**: Message rejected by gateway or telecom carrier.
   - **`DELIVERED`**: Successfully delivered to handset.
   - **`EXPIRED`**: Handset was powered off or out of range beyond carrier retry window.

## 🚨 Troubleshooting Common Issues

### 1. Insufficient Balance Exception

```json
{
  "message": "Itinarea balance not enough",
  "status_code": 402,
  "errors": {
    "company_id": 1,
    "channel_id": "sms"
  }
}
```

**Resolution**: Ensure your company has active credits assigned to the `sms` channel or a global unrestricted credit balance.

### 2. Invalid Mobile Number Format

If a recipient's phone number is malformed (e.g. missing country code or invalid digits), the campaign initialization will bypass or mark that recipient as failed during delivery log creation.

**Resolution**: Standardize all recipient phone numbers in **E.164 format** (e.g., `212671791198`).
