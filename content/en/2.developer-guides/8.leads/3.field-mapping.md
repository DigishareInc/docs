This guide is for lead form creators (Facebook/Meta Ads, TikTok Ads) to ensure that fields are correctly mapped to the Digishare CRM attributes.

## 🚀 How it Works

When you create a lead form, the **Field Label** you choose determines where the data is stored in Digishare. You can use **English**, **French**, or **Arabic** labels — Digishare handles them all automatically.

::callout{type="info"}
Digishare uses a normalization process to ensure that case sensitivity, spacing, and accents (like in French) do not prevent data from being correctly captured.
::

## 📋 Supported Fields

| CRM Field | Accepted Labels |
| :--- | :--- |
| **Full Name** | `full_name`, `name`, `nom_complet`, `الاسم_الكامل`, `الاسم_بالكامل`, `الاسم`, `اسم` |
| **First Name** | `first_name`, `prénom`, `الاسم_الأول`, `الاسم_الشخصي`, `الإسم_الشخصي` |
| **Last Name** | `last_name`, `nom`, `family_name`, `nom_de_famille`, `الاسم_العائلي`, `الإسم_العائلي`, `النسب`, `اللقب` |
| **Email** | `email`, `e_mail`, `courriel`, `البريد_الإلكتروني`, `الايميل` |
| **Phone** | `phone_number`, `phone`, `telephone`, `numéro_de_téléphone`, `رقم_الهاتف`, `الهاتف`, `رقم_الواتساب`, `جوال`, `الموبايل` |
| **Language** | `lang`, `language`, `language_code`, `اللغة` |
| **City** | `city`, `ville`, `town`, `المدينة` |
| **Address** | `address`, `adresse`, `street`, `rue`, `العنوان` |
| **Gender** | `gender`, `sex`, `sexe`, `الجنس` |
| **Birthday** | `birthday`, `dob`, `date_of_birth`, `birthdate`, `تاريخ_الميلاد` |
| **Job Title** | `job_title`, `title`, `work`, `profession`, `المهنة`, `الوظيفة` |
| **Company** | `company`, `company_name`, `société`, `entreprise`, `الشركة`, `المؤسسة` |

## 💡 Important Notes

### 1. Case, Spacing & Accents

Digishare normalizes labels automatically. For example:
- `Full Name` and `full name` both match `full_name`.
- `Prénom` (with accent) matches `prenom` (without accent).

::callout{type="success"}
You don't need to worry about the exact formatting of labels in your lead forms as long as they are among the supported ones above.
::

### 2. Custom Fields

Any label not listed above will be automatically saved under the `custom_fields` attribute in the CRM. **No data is ever lost** — even if the field is not pre-defined in our mapping system.
